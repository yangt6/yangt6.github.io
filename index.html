LDAP基础学习

LDAP：Lightweight Directory Access Protocol，轻量目录访问协议,他主要用做用户信息查询（如邮箱、电话等）或对各种服务访问做后台认证以及用户数据权限管控。

名词解释
DC：domain component一般为公司名，例如：dc=163,dc=com
OU：organization unit为组织单元，最多可以有四级，每级最长32个字符，可以为中文
CN：common name为用户名或者服务器名，最长可以到80个字符，可以为中文
DN：distinguished name为一条LDAP记录项的名字，有唯一性，例如：dc:”cn=admin,ou=developer,dc=163,dc=com”

–env PHPLDAPADMIN_LDAP_HOSTS=ldap.example.com \
–detach osixia/phpldapadmin:0.7.2

拉取PHPLDAPADMIN镜像并且映射到端口6443

docker初次接触

—什么是docker？

Docker可以认为是一种先进的容器技术。 docker它包含三个基本概念：镜像，容器和仓库
镜像是一个特殊的文件系统，相当于一个沙盒环境，它提供了容器运行的程序，库，资源，配置等。

容器的实质是进程，容器进程运行于属于自己的独立的 命名空间。

仓库用于存放镜像文件，通常，一个仓库会包含同一个软件不同版本的镜像，而标签就常用于对应该软件的各个版本。我们可以通过 <仓库名>:<标签> 的格式来指定具体是这个软件哪个版本的镜像。

安装docker

执行命令：
$ curl -fsSL get.docker.com -o get-docker.sh
$ sudo sh get-docker.sh –mirror Aliyun

启动docker命令：
$ sudo systemctl enable docker
$ sudo systemctl start docker

测试 Docker 是否安装正确：
$ docker run hello-world
执行代码：
Hello from Docker!
。。。。/略

成功

可以使用docker images查看

接下来尝试使用docker搭建一个nginx的web服务器
docker search nginx #查询nginx相关镜像文件
docker pull nginx #拉取镜像文件到本地
docker run –name webserver -d -p 80:80 nginx #启动nginx服务器，并且映射了80端口
打开浏览器输入自己的ip地址就可以访问了
